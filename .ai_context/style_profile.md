# 写作风格指纹 / Style Profile

> 本文件由 Style Extractor 根据 `资料/` 中的文章自动生成，用于指导后续写作智能体的风格迁移。  
> （示例：将本文件复制到论文项目根目录的 `.ai_context/` 使用）

## Core Style DNA

- **Tone（语气）**: 客观中立的学术论述风格。采用第三人称视角（本设计、本文、本研究）进行表述，避免第一人称。语气严谨、克制，强调逻辑递进与证据支撑。
- **Sentence Pattern（句式）**: 以中长句为主，平均句长约 25–35 字。善用逻辑连接词（因此、由此可见、在此基础上、然而、此外、首先、其次、最后）串联论证。偏好"问题—方法—结果"的递进结构。
- **学术严谨度（1–10 分）**: 8 分。词汇以领域术语与规范表达为主，兼顾可读性与专业性。技术术语首次出现时给出英文缩写（如 Lightmap、VLM、PRT），后续可简化使用。

## Do's（我要的风格）

- 使用「本设计」「本文」「本研究」作为主语，避免使用「我」「我们」进行自述。
- 在引入新概念时，先给出中文全称，再标注英文缩写，如：全局光照（Global Illumination，简称 GI）。
- 采用分点枚举结构组织论述，如：（1）（2）（3）或「首先……其次……最后……」。
- 图表引用使用规范格式：如图 X 所示、如表 X 所示。
- 公式后紧跟变量说明，使用「其中，……表示……」的句式。
- 段落间使用过渡句承上启下，如「在此基础上」「针对上述问题」「综上所述」。
- 强调贡献与创新时使用「提出」「设计」「实现」等动词，避免夸张表述。
- 实验与结论部分使用「实验结果表明」「相较于……」「验证了……」等客观表述。
- 保持中英文术语混排时的格式一致：中文为主，英文缩写置于括号内补充。
- 章节标题采用「第 X 章 XXX」与「X.X XXX」的层级结构。

## Don'ts（我不要的风格）

- 拒绝使用典型的 AI 味高频词：深入探讨、 delve into、utilize、tapestry、landscape、realm、plethora。
- 避免空洞的过渡语：值得一提的是、无独有偶、毋庸置疑。
- 不使用口语化表达：搞定、搞定、搞定、非常非常、特别特别。
- 避免过度形容词堆砌：革命性的、颠覆性的、前所未有的（除非有明确支撑）。
- 不滥用「可以」「能够」的重复，注意句式变化。
- 拒绝使用「众所周知」「显而易见」等主观判断式开头。
- 避免冗长的从句嵌套，保持句子的可读性。
- 不使用「etc.」「等」在列举不完整时草率收尾，应明确给出代表性示例。
- 图表引用不说「见下图」「如上所示」，须明确「如图 X 所示」「如表 X 所示」。
