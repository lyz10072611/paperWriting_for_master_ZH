## 二、论文项目结构

以**论文中文名称**为根目录：

```
<论文中文名称>/
├── main.tex
├── chapters/           # 各章 .tex
├── .ai_context/        # 风格、错题本、记忆（从本 Skill 复制模板）
│   ├── style_profile.md
│   ├── error_log.md
│   ├── custom_specs.md
│   └── memory/
└── 资料/
    ├── 01_格式要求/     # 毕业论文格式规范 PDF
    ├── 02_参考毕业论文/ # 他人已通过论文
    ├── 03_我的内容/     # 中期报告、初审稿等
    ├── 04_我的小论文/   # 小论文（可扩写为毕业论文）
    └── 提取/            # parse_pdf 输出
```

详见 [project_structure.md](.ai_context/project_structure.md)。

## 三、初始化步骤

1. 以论文中文名称为文件夹名创建项目根目录
2. 创建 `chapters/`、`资料/01_格式要求/`、`资料/02_参考毕业论文/`、`资料/03_我的内容/`、`资料/04_我的小论文/`、`资料/提取/`
3. 将本 Skill 的 `.ai_context/` 复制到项目（含 style_profile、error_log、memory、custom_specs 等模板）
4. 放入资料并执行 Format Spec Reader、Style Extractor

## 五、硕士论文格式规范

### 5.1 章节结构（参考）

```
摘要（中英文）
第1章 绪论
  1.1 研究背景与意义
  1.2 国内外研究现状
  1.3 研究内容与方法
  1.4 论文结构安排
第2章 相关理论与技术
第3章 （方法/模型/算法）设计
第4章 实验与结果分析
第5章 系统设计与实现（若适用）
第6章 总结与展望
参考文献
致谢
附录（若需要）
```

### 5.2 引用与图表

- **引用格式**：一般采用 GB/T 7714，正文用「作者（年份）」或「[序号]」
- **图题**：置于图下方，格式如「图 3-1 某方法流程图」
- **表题**：置于表上方，格式如「表 4-1 实验对比结果」
- **正文引用**：用「如图 3-1 所示」「由表 4-1 可知」，避免「见下图」「见下表」

### 5.3 语言规范

- 使用规范学术用语，避免网络用语、口语化表述
- 数字与单位：如「3.2%」「10 mm」「256×256 像素」
- 首次出现缩写需全称，如「支持向量机（Support Vector Machine, SVM）」

### 5.4 方法与数据章节写作要求

- **方法段优先保证可解释性**：去 AI 味不能以牺牲技术说明为代价
- **保留因果链**：像「由于…因此…」「考虑到…本文…」这类句式，只要承担真实逻辑功能，不应机械删除
- **保留步骤目的**：对于「辐射定标用于…」「大气校正用于…」这类说明，若其作用是解释处理目的，应保留或改写为同等清晰的表达，不得压缩为仅剩动作陈述
- **保留复现信息**：数据来源、产品级别、分辨率、样本筛选条件、数量、区域独立性等内容不得因润色而弱化
- **避免过度压缩**：方法段允许比摘要、结论更详细；优先删套话、空话、重复话，不优先删技术解释
- **审稿人视角**：润色后应让非本方向评审也能看懂“为什么这样做、做了什么、得到什么”

## 六、去 AI 味：reduce-ai_ZH 规则

写作或润色时须遵守以下规则。**完整 24 种 AI 写作模式**详见 [reduce-ai_ZH_patterns.md](.ai_context/reduce-ai_ZH_patterns.md)。

### 6.1 核心规则速查

1. **删除填充短语** - 去除开场白和强调性拐杖词
2. **打破公式结构** - 避免二元对比、戏剧性分段、修辞性设置
3. **变化节奏** - 混合句子长度。两项优于三项。段落结尾要多样化
4. **信任读者** - 直接陈述事实，跳过软化、辩解和手把手引导
5. **删除金句** - 如果听起来像可引用的语句，重写它

### 6.2 学术场景下「个性与灵魂」的调整

- **人称**：学术论文统一使用「本文」「本研究」「本设计」作主语，禁用「我」「我们」
- **第一人称**：非正式文体可「适当使用我」，中文学位论文正文禁用
- **语气**：客观克制，不做夸张评价，结论须有文献或实验支撑

### 6.3 中文学术论文特有禁用表达

| 禁用表达 | 典型问题 |
|----------|----------|
| 综上所述、总之、简而言之 | 机械式总结 |
| 值得注意的是、值得一提的是 | 高频 AI 套话 |
| 在当今时代、随着…的不断发展 | 空洞开场 |
| 具有重要意义、具有广阔的应用前景 | 万能废话 |
| 深入探讨、深入研究、进一步分析 | 空洞动词堆砌 |
| 众所周知、显而易见 | 假权威语气 |
| 从某种程度上、在一定程度上 | 过度 hedging |
| 本文旨在、本研究的目的是 | 可在摘要用一次，正文避免重复 |
| …的背景下，…成为了… | 模板化过渡 |

### 6.4 学术论文禁用结构模式

- **禁止**每段都以「首先…其次…再次…最后」或「一方面…另一方面」机械展开
- **禁止**连续多段使用相同句式（如「…是…的关键」「…对…具有重要作用」）
- **禁止**每句都是「主语+是+宾语」或「由于…因此…」的固定搭配
- **禁止**过度使用「通过…，实现了…」「基于…，提出了一种…」的模板句

### 6.5 学术写作替代策略

- 用**具体事实或数据**替代抽象评价
- 用**因果链条**替代空泛过渡：先给现象/数据，再给解释，最后给结论
- 用**短句+长句交错**替代单一节奏
- **一段一事**：每段聚焦一个论点
- **主谓宾清晰**：少用「的」字长串
- **术语一致**：全文同一概念用同一术语
- **先删套话，再动解释**：优先删除「具有重要意义」「广阔应用前景」等空泛表达，不随意压缩承担技术说明功能的句子
- **可改写，不硬删**：像「用于」「考虑到」「为避免」若承担解释功能，可换成更自然的说法，但不应整体删除

### 6.6 润色检查清单

- [ ] 无「综上所述」「值得注意的是」等套话
- [ ] 无连续 3 句以上相同句式
- [ ] 段落有长有短，过渡自然
- [ ] 结论有依据（数据/文献/逻辑）
- [ ] 术语全文统一
- [ ] 符合学校或期刊的格式要求（若用户已提供）
- [ ] 方法段中的“处理步骤-作用-目的”是否仍然完整
- [ ] 数据段中的来源、尺度、筛选条件、样本数量是否仍然完整

### 6.7 学术论文改写示例

**AI 味：** 综上所述，本研究提出了一种基于深度学习的识别方法，该方法具有重要的理论和实践意义，为相关领域的发展提供了新的思路。

**人类风格：** 本文方法在测试集上 mAP 达 89.2%，相较基线提升 4.7 个百分点。结果表明，引入注意力机制能够有效缓解小目标漏检问题。
